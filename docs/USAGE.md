## 1. 产品概述

Sun-Panel 企业版是一款面向团队/企业的导航面板与浏览器首页，支持：

- **公开首页**：未登录用户可浏览公开导航分组（正常设计，无需修改）。
- **RBAC 权限控制**：基于角色+部门的细粒度权限管理。
- **部门隔离**：资源按部门归属，配合角色实现数据权限。
- **审计与日志**：操作日志、登录日志、定时任务、通知公告。
- **安全增强**：登录失败锁定、2FA 双因素认证、IP 登录白名单、HTTPS/TLS 1.2+。
- **便签与中转站**：桌面便签、加密粘贴板（阅后即焚、密码保护）。
- **一键获取图标**：自动从目标站点 / 公共图标服务获取 favicon。

## 3. 功能模块说明

### 3.1 首页与导航

- **公开模式**：未登录用户可看到标记为「公开」的分组/图标。
- **登录模式**：登录后显示个人/部门/系统级图标，并出现「系统状态」小部件。
- **分组管理**：在「系统应用 → 分组管理」中新增、编辑、删除图标分组。
- **图标添加**：
  - 输入名称、URL、内网 URL（可选）、打开方式。
  - 点击「获取图标」自动从目标站点获取 favicon（失败后自动使用 Google / DuckDuckGo / icon.horse 等公共图标服务）。
  - 支持本地上传、在线图标库、SVG 图标。

### 3.2 账号管理（RBAC）

- 路径：系统应用 → 账号管理。
- 功能：新增、编辑、删除、导入/导出用户。
- 字段：账号、昵称、密码、角色、部门、状态。
- **注意**：账号之间数据不互通；删除账号后其资源保留但失去归属。

### 3.3 角色权限（RBAC）

- 路径：系统应用 → 角色权限。
- 功能：
  - 预置角色：超级管理员（role=1）、部门管理员、普通用户。
  - 自定义角色：可勾选菜单权限与按钮权限。
  - 超级管理员默认拥有所有权限。
- **权限生效**：首页「系统应用」菜单根据当前用户角色动态渲染。

### 3.4 部门管理

- 路径：系统应用 → 部门管理。
- 功能：
  - 树形部门结构。
  - 每个图标/资源可归属到部门。
  - 用户继承所在部门，角色+部门共同决定可见资源。
- **联动规则**：
  - 删除部门时，原部门用户部门 ID 置空，图标需要管理员重新分配。
  - 变更用户角色/部门后，需重新登录或刷新页面以生效。

### 3.5 系统设置

- 路径：系统应用 → 系统设置。
- 包含：
  - Logo 设置
  - 背景设置（上传、自定义 URL、预设）
  - 登录配置（验证码、IP 白名单）
  - 通知管理
  - 备份管理
  - 操作日志 / 登录日志
  - 定时任务

### 3.6 备份管理

- 路径：系统应用 → 系统设置 → 备份。
- 功能：
  - 手动创建 SQLite 备份。
  - 定时任务自动备份（Cron 表达式）。
  - 保留天数策略。
  - 恢复：选择备份文件覆盖当前数据库（谨慎操作）。
  - 导出/导入：用于迁移或离线保存。

### 3.7 通知管理

- 路径：系统应用 → 系统设置 → 通知管理。
- 类型：公告、站内信。
- 展示位置：登录页、首页。
- 目标用户：指定用户 ID（逗号分隔），留空为全员。
- 一次性阅读：用户点击「知道了」后不再显示。

### 3.8 操作日志 / 登录日志

- 操作日志：记录管理员/用户的增删改查操作、路径、IP、耗时、状态码。
- 登录日志：记录登录成功/失败、IP、浏览器 UA、锁定原因。
- 支持按关键词、时间范围筛选。

### 3.9 定时任务

- 路径：系统应用 → 系统设置 → 定时任务。
- 功能：
  - 按 Cron 表达式执行提醒或备份。
  - 支持启动、暂停、立即执行、删除。
  - 查看执行日志。
- 示例 Cron：`0 0 2 * * ?`（每天凌晨 2 点）。

### 3.10 便签（Sticky Note）

- 入口：首页右侧悬浮的绿色「+」按钮。
- 功能：
  - 新建、拖拽、缩放、改色、停用/启用、删除。
  - 内容自动保存（防抖 500ms）。
  - 支持透明模式（在系统设置中开启）。
- 使用提示：
  - 鼠标拖拽便签顶部移动。
  - 右下角拖拽改变大小。
  - 顶部彩色圆点切换颜色。
  - 眼睛图标停用/启用，关闭图标删除。

### 3.11 中转站（Paste Bin）

- 路径：系统应用 → 中转站。
- 功能：
  - 创建文本/文件中转。
  - 访问密码保护。
  - 访问次数限制。
  - 过期时间（1 小时 ~ 7 天）。
  - 阅后即焚：被读取 1 次后自动销毁。
- 使用：创建后复制访问码或链接分享给他人。

## 4. 权限控制说明

### 4.1 角色权限矩阵

| 角色 | 权限范围 |
|------|----------|
| 超级管理员 | 全部功能，不受部门限制 |
| 部门管理员 | 管理部门内用户、资源 |
| 普通用户 | 仅查看/编辑被授权的资源 |

### 4.2 首页公开显示

- 首页未登录显示公开内容属于正常设计。
- 在图标编辑中勾选「公开」后，未登录用户即可看到该图标。
- 管理功能必须登录并具备相应角色权限。

### 4.3 登录安全

- **登录失败锁定**：连续失败 5 次锁定 30 分钟（可在代码中配置）。
- **2FA**：在用户个人信息中绑定 TOTP（如 Google Authenticator）。
- **IP 白名单**：系统设置 → 登录配置中设置允许登录的 IP 列表。

## 5. 一键获取图标说明

### 5.1 工作原理

1. 解析目标站点 HTML，提取 `<link rel="icon" ...>` 标签。
2. 若未找到，尝试默认路径：`/favicon.ico`、`/favicon.png`、`/apple-touch-icon.png`。
3. 仍未成功则使用公共图标服务兜底：
   - DuckDuckGo Icons
   - Google Favicon Service
   - icon.horse
   - Clearbit Logo
4. 下载成功后将图标保存到服务器本地，返回本地 URL。

### 5.2 失败处理

- 部分站点有反爬或 CORS 限制，可能仍无法获取。
- 失败后可手动上传图标或粘贴图标 URL。
- 公共图标服务受网络环境影响，可重试或换用其他图标。

## 7. 常见问题

### Q1：页面加载后一直转圈/JS 报错
- 原因：旧版本 PWA 缓存或服务端静态资源不一致。
- 解决：强制刷新浏览器（Ctrl+F5 或 Cmd+Shift+R），或清除浏览器缓存。

### Q2：图标获取失败
- 检查目标 URL 是否可访问。
- 尝试手动上传图标。
- 检查服务器是否能访问外网（公共图标服务需要）。

### Q3：修改用户角色/部门后权限未生效
- 权限缓存基于登录 token，需要重新登录或刷新页面。

### Q4：如何更新版本
- 更新本地源码 → 推送 GitHub → 服务器拉取并构建 → 部署容器。
- 或触发 GitHub Actions 自动构建 Docker 镜像并推送到 Docker Hub。
